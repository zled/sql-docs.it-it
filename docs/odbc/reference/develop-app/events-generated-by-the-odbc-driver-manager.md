# <a name="events-generated-by-the-odbc-driver-manager"></a>Eventi generati da Gestione Driver ODBC
> [!IMPORTANT]  
>  Supporto per Visual Studio Analyzer è stata rimossa a partire da Windows 8 (solo incluso Visual Studio Analyzer in versioni precedenti di Visual Studio). Per un'alternativa meccanismo di risoluzione dei problemi, utilizzare la traccia dell'offerta.  
  
 Quando si fa clic sul pulsante Avvia Visual Studio Analyzer, vengono registrati gli eventi generati da Gestione Driver ODBC. Lo strumento stesso offre gli eventi definiti dal sistema e la possibilità di creare eventi personalizzati. Per ulteriori informazioni sugli eventi, vedere il *Guida di riferimento di Visual Studio Analyzer* all'interno del gruppo di Visual Studio della documentazione.  
  
|Eventi di Visual Studio Analyzer|Description|  
|----------------------------------|-----------------|  
|**Chiamata**|Generato su ogni voce dell'API ODBC.|  
|**ReturnException**|Se il codice restituito SQL_ERROR, generato in tutte le API ODBC.|  
|**ReturnNormal**|Se il codice restituito non è SQL_ERROR, generato in tutte le API ODBC.|  
|**Avvio di connessione**|Indica l'avvio di una connessione; generato quando Gestione Driver ODBC chiama l'API di connessione del driver.|  
|**Connessione completata**|Indica il completamento di una connessione; generato quando la connessione del driver API tornare a gestione Driver ODBC.|  
|**Disconnettere inizio**|Generato quando Gestione Driver ODBC chiama il driver **SQLDisconnect** (funzione).|  
|**Connessione chiusa**|Generato quando il driver **SQLDisconnect** funzione a gestione Driver ODBC.|  
|**QuerySend**|Generato quando Gestione Driver ODBC chiama il driver **SQLPrepare**, **SQLExecute**, **SQLExecDirect** funzioni, nonché le funzioni di catalogo, ad esempio **SQLTables** e **SQLColumns**.|  
|**QueryResult**|Generato quando il driver restituisce un set di gestione Driver ODBC per le funzioni che includono query di risultati.|  
|**TransactionStart**|Generato quando un'applicazione imposta il valore SQL_ATTR_AUTOCOMMIT su SQL_AUTOCOMMIT_OFF o dopo un'applicazione chiama correttamente **SQLEndTran**.|  
|**TransactionCommit**|Generato quando un'applicazione chiama **SQLEndTran** per eseguire il commit di una transazione locale.|  
|**TransactionRollback**|Generato quando un'applicazione chiama **SQLEndTran** per eseguire il rollback di una transazione locale.|  
|**JoinDTC**|Generato quando un'applicazione viene aggiunto a Distributed Transaction Coordinator (DTC).|  
|**LeaveDTC**|Generato quando un'applicazione abbandona Distributed Transaction Coordinator (DTC).|
